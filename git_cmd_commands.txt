GIT CMD COMMANDS FOR PUSHING BRANCH
====================================

Type these commands in the Git CMD window that just opened:

1. NAVIGATE TO PROJECT DIRECTORY:
   cd C:\Users\PHealy\CascadeProjects\windsurf-project\Predicted-Feeder-Congestion

2. CHECK CURRENT STATUS:
   git status

3. ADD ALL FILES:
   git add .

4. COMMIT CHANGES:
   git commit -m "Add CALMAC load data integration and merge functionality

- Updated data_wrangling.py with load data integration
- Added load_data_loader.py for hourly load processing  
- Created exploration and demonstration scripts
- Added comprehensive documentation
- Integrated Res_GP_Elec_2024.csv merge pipeline"

5. CHECK BRANCH:
   git branch

6. PUSH TO GITHUB:
   git push origin Merge-CALMAC-Load-Shape

7. VERIFY PUSH:
   git status

IF YOU SEE AUTHENTICATION ERRORS:
=================================

For HTTPS authentication:
- Username: Your GitHub username
- Password: Personal Access Token (NOT your GitHub password)

To get Personal Access Token:
1. Go to GitHub.com → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token" → "Generate new token (classic)"
3. Select scopes: Check "repo" (full control of private repositories)
4. Click "Generate token"
5. Copy the token (you won't see it again)
6. Use this token as your password when Git prompts

TROUBLESHOOTING:
===============

If push fails with "remote: Repository not found":
- Check if you have access to the repository
- Verify the repository URL is correct

If push fails with "authentication failed":
- Generate a new Personal Access Token
- Make sure token has 'repo' permissions

If push fails with "remote: Permission denied":
- You may not have write access to this repository
- Contact repository owner for permissions
